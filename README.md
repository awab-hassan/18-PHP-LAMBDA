# php-lambda-bref-scylladb

A proof-of-concept AWS Lambda function written in PHP 8.1 using [Bref](https://bref.sh/), connecting to a self-hosted ScyllaDB cluster over native CQL and returning query results as JSON. Built to validate PHP-on-Lambda connectivity to ScyllaDB before committing to the approach.

## How It Works

1. Lambda cold-start loads `index.php`; `vendor/autoload.php` initialises Bref and the Cassandra PHP driver.
2. The handler opens a session via `Cassandra::cluster()->withContactPoints(...)->build()->connect('keyspace1')`.
3. Executes `SELECT * FROM standard1 LIMIT 1` and collects rows into an array.
4. Returns `statusCode: 200` with the row data on success, or `statusCode: 500` with the error string on `Cassandra\Exception`.

## Architecture

```
Caller / API
    |
    v
Lambda: bref-php-function-dev
(PHP 8.1, provided.al2, 1024 MB, 6s timeout)
    |
    | CQL port 9042
    v
ScyllaDB cluster (self-hosted)
    |
    v
{ statusCode: 200, body: { rows: [...] } }
```

## Prerequisites

- AWS SAM CLI
- Composer installed locally
- Network path from Lambda to ScyllaDB on port 9042 (VPC peering or public IP with security group allowance)
- A Bref-compatible Cassandra extension layer (the stock `php-81` Bref layer does not include it — use a custom or community layer)

## Deployment

```bash
cd existing-function
composer install --no-dev --optimize-autoloader
cd ..
sam build -t cf.yaml
sam deploy --guided
```

## Notes

- The contact point is a public IP. For any real use, move ScyllaDB behind VPC peering or PrivateLink and restrict the security group to the Lambda ENI.
- `keyspace1` / `standard1` is the default schema generated by `cassandra-stress`. This handler was a connectivity smoke test, not a business query.
- ScyllaDB is wire-compatible with Cassandra, so the DataStax PHP driver works against it without modification.
- The SAM template targets `eu-west-1`. The Bref layer ARN (`php-81:103`) is region-specific and must be updated for other regions.
